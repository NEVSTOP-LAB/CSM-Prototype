我需要设计一对安全字符串的转换函数，可以将字符串中的特殊字符替换为HEXCODE, 作为安全字符串，在应用中使用；并提供安全字符串转换回普通字符串的功能。

# 设计目标

1. **可逆**：任意输入字符串都能 100% 恢复。
2. **安全**：输出仅包含受控字符集合，便于在系统中传输和存储。
3. **高效**：单次线性扫描，时间复杂度 O(n)。
4. **全面**：覆盖 Unicode、控制字符、空字符、emoji、lone surrogate 等边界场景。

# 编码算法

## 1) 输入层

将 Python `str` 先转换为 UTF-8 字节流，使用 `errors="surrogatepass"`，确保连非常规 Unicode（例如 lone surrogate）也可稳定往返。

## 2) 输出字符策略

- 允许直接输出的“安全字符”（unreserved）：`[A-Za-z0-9._-]`
- 其余字节全部转义为：`~HH`
  - `~` 为转义前缀
  - `HH` 为两位十六进制（大写）

例如：
- 空格 `0x20` -> `~20`
- `~` 本身 `0x7E` -> `~7E`
- 中文字符先转 UTF-8 多字节，再逐字节编码

## 3) 解码策略

按字符线性扫描：
- 若遇到 `~`，必须紧跟两位十六进制，组合为一个字节
- 若是普通字符，必须属于 `[A-Za-z0-9._-]`
- 其他输入判为非法并抛出 `ValueError`

最终将字节流按 UTF-8 + `surrogatepass` 解码回原字符串。

# 为什么该方案无歧义

- `~` 不在直出字符集合中，因此不会与普通字符混淆。
- 所有 `~` 都是转义起点，且固定长度 3（`~HH`）。
- 解码器可严格校验格式，避免 silent corruption。

# 复杂度与效率

- 编码：单次遍历字节，O(n)
- 解码：单次遍历字符，O(n)
- 仅使用轻量字符串拼接列表与 `bytearray`，内存开销可控

# 覆盖的 corner cases

1. 空字符串
2. 纯安全字符（不应被改写）
3. 含空格和常见特殊符号（应转 HEX）
4. 转义前缀字符 `~` 本身
5. 控制字符：`\n`、`\t`、`\x00`
6. Unicode：中文、emoji
7. 超长混合字符串
8. lone surrogate（`\ud800`）
9. 全字节范围 roundtrip（0x00-0xFF 映射到 `latin1` 字符）
10. 非法输入处理：不完整转义、非十六进制、非法直出字符
11. 入参类型错误（非 `str`）
